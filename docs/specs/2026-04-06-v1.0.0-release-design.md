# Strimma v1.0.0 Release Design

**Date:** 2026-04-06
**Status:** Partially implemented — Track 1 (polish) delivered, Track 2 (wallpaper) rejected

> **Resolution:** Track 2 was implemented and tested but rejected. No public API exists for custom AOD content on stock Android. The live wallpaper approach replaces the user's wallpaper for zero added value — notification (lock screen), widget (home screen), and BG icon (AOD) already cover all three surfaces. See `ideas.md` Rejected section.

## Track 1: Polish

### 1.1 Fix Dual-Storage Atomicity

**File:** `data/SettingsRepository.kt`

`setGlucoseSource()` (line ~295) and `setStartOnBoot()` (line ~361) write to DataStore then SharedPreferences sequentially. A crash between the two writes leaves `GlucoseNotificationListener` reading stale data from SharedPreferences via `getGlucoseSourceSync()`.

**Fix:** Move the SharedPreferences write inside the `dataStore.edit {}` block so the sync write only happens if DataStore commits successfully.

### 1.2 NightscoutPusher Tests

**File to test:** `network/NightscoutPusher.kt`

Zero tests exist for the core push reliability path. Write tests covering:

- Successful push on first attempt
- Retry on HTTP failure (up to MAX_RETRY_ATTEMPTS)
- Linear backoff timing (attempt * 5s, capped at 60s)
- CancellationException rethrown (not swallowed)
- Unpushed readings recovered after restart (marked `pushed=0`)
- Batch push of multiple pending readings
- PushFailureTracker integration (alert after sustained failures)

Use Ktor MockEngine for HTTP mocking. Follow existing test patterns (flat tests, no nesting, JUnit 4 + Robolectric).

### 1.3 AlertManager Core Logic Tests

**File to test:** `notification/AlertManager.kt`

Only the pause feature has tests (AlertManagerPauseTest, 21 tests). Core alert logic is untested. Write tests covering:

- Each alert type fires at its threshold (urgent low, low, high, urgent high, stale)
- Alerts don't fire when value is in range
- Alerts don't fire when snoozed
- Correct notification channel per alert type
- Stale alert fires when no reading within stale threshold
- Predictive alerts ("low soon", "high soon") fire when prediction crosses threshold
- Alert clears when condition resolves

### 1.4 Setup Wizard Spec Alignment

The setup wizard spec (`docs/specs/2026-03-25-setup-wizard-design.md`) describes 6 steps including a Nightscout push configuration step. The implementation has 5 steps (no Nightscout step). Nightscout configuration happens in Settings instead.

**Action:** Update the spec to match the implementation. The 5-step wizard is the correct design — Nightscout push is optional and complex enough to belong in Settings, not the first-run flow.

## Track 2: AOD / Lock Screen Live Wallpaper

### Problem

People want to see their BG without unlocking their phone. The existing notification shows BG in the status bar icon and in the notification shade, but:
- The status bar icon is tiny (24dp) and hard to read
- On AOD, only notification icons are visible — not the expanded notification
- Users must swipe/tap to see the graph or delta

GlucoDataHandler solves this with a live wallpaper. Strimma should too.

### Design

A `WallpaperService` that renders BG data directly on the lock screen and AOD.

#### What It Shows

```
          (lock screen / AOD)

              13.5 ↘
             -1.1 mmol/L
              2 min ago

        ┌─────────────────┐
        │  ····           │
        │       ·····     │  (optional mini graph, 1h)
        │            ···  │
        └─────────────────┘
```

- **BG value** — large, centered, status-colored (cyan/amber/coral)
- **Trend arrow** — next to BG value
- **Delta** — below BG, smaller, secondary color
- **Time since reading** — below delta, tertiary color
- **Mini graph** — optional, last 1 hour, below text, using existing graph rendering logic
- **Background** — transparent (system handles AOD dimming)

#### OLED Considerations

AOD on OLED displays lights individual pixels. Minimize lit area:
- No background fill — transparent
- Thin text, no glow/shadow effects
- Mini graph uses dots only, no connecting lines, no fill
- Status colors are already high-contrast on black (cyan, amber, coral)

#### Architecture

```
Room DB (latestFlow())
  → BgWallpaperService.BgWallpaperEngine
    → Canvas draw on SurfaceHolder
```

- `BgWallpaperService` extends `WallpaperService`
- Inner `BgWallpaperEngine` extends `Engine`
- Gets `ReadingDao` and `SettingsRepository` via Hilt `EntryPointAccessors.fromApplication()`
- Collects `readingDao.latestFlow()` in a `CoroutineScope`
- Redraws canvas on each new reading emission
- Pauses collection when `onVisibilityChanged(false)` (saves battery when not visible)
- Resumes when `onVisibilityChanged(true)`

#### Files to Create

| File | Purpose |
|------|---------|
| `wallpaper/BgWallpaperService.kt` | WallpaperService + Engine with Canvas rendering |
| `wallpaper/WallpaperEntryPoint.kt` | Hilt EntryPoint interface for DI into WallpaperService |
| `res/xml/bg_wallpaper.xml` | Wallpaper metadata (thumbnail, description) |
| `wallpaper/BgWallpaperRenderer.kt` | Rendering logic (BG value, arrow, delta, graph) — testable |

#### Files to Modify

| File | Change |
|------|--------|
| `AndroidManifest.xml` | Register BgWallpaperService with BIND_WALLPAPER permission |
| `res/values/strings.xml` | Add wallpaper description string (all locales) |
| `ui/SettingsScreen.kt` | Add "Live wallpaper" toggle in Display section |
| `data/SettingsRepository.kt` | Add wallpaper enabled + show graph on wallpaper settings |

#### Settings

In Settings > Display:
- **Live wallpaper** toggle — when tapped, opens Android's live wallpaper picker with Strimma pre-selected
- **Show graph on wallpaper** toggle — controls whether the mini graph renders (default: on)

#### User Flow

1. User enables "Live wallpaper" in Strimma Settings > Display
2. Android's wallpaper picker opens, showing Strimma preview
3. User selects "Set wallpaper" → chooses "Lock screen" (recommended) or "Home and lock screen"
4. BG appears on lock screen immediately, AOD on next screen-off

#### Testing

- `BgWallpaperRendererTest` — unit tests for rendering logic (value positioning, color selection, staleness handling, graph dot placement)
- Rendering logic separated into `BgWallpaperRenderer` (pure Canvas drawing, testable) vs. service lifecycle (in `BgWallpaperService`)

- New guide page: `docs/guide/lock-screen.md` — setup instructions, screenshots placeholder, FAQ
- Update `mkdocs.yml` nav to include the new page
- Update `docs/faq.md` with AOD/lock screen Q&A

## Out of Scope

- Caregiver/remote following (future — needs relay infrastructure design)
- Watch support (no hardware)
- Home screen wallpaper customization (users can choose lock screen only via Android picker)
- Wallpaper settings activity (use Android system picker, not a custom activity)

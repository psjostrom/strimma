# Strimma i18n Design

## Summary

Extract all user-visible hardcoded strings (~165) into Android resource files. Support English (default), Swedish, Spanish, French, and German. No third-party libraries — uses Android's built-in resource system.

## Languages

- **English** — default locale (`values/strings.xml`)
- **Swedish** — `values-sv/strings.xml`
- **Spanish** — `values-es/strings.xml`
- **French** — `values-fr/strings.xml`
- **German** — `values-de/strings.xml`

Locale defaults to the system setting. An in-app language picker in Display settings allows overriding the locale at runtime via Android's `LocaleManager` API (API 33+). Supported locales are declared in `res/xml/locales_config.xml` and referenced from the manifest via `android:localeConfig`.

## Technical Approach

### Compose UI

Replace all hardcoded strings with `stringResource(R.string.key)`:

```kotlin
// Before
Text("Statistics")

// After
Text(stringResource(R.string.stats_title))
```

For strings with arguments, use format parameters:

```kotlin
// Before
Text("Low in $minutes min")

// After
Text(stringResource(R.string.main_prediction_low, minutes))
```

### Non-Compose Code (notifications, alerts, service)

Use `context.getString(R.string.key)` — same resource files, different access pattern:

```kotlin
// Before
builder.setContentTitle("Urgent Low")

// After
builder.setContentTitle(context.getString(R.string.alert_urgent_low_title))
```

### Glance Widget

The widget uses Jetpack Glance, not regular Compose. `stringResource()` is **not available** in Glance composables. Use `LocalContext.current` inside `provideContent {}` to resolve strings, then pass them down:

```kotlin
override suspend fun provideGlance(context: Context, id: GlanceId) {
    provideContent {
        val ctx = LocalContext.current
        WidgetContent(
            noDataText = ctx.getString(R.string.widget_no_data),
            // ...
        )
    }
}
```

### Enum Labels

`GlucoseSource`, `InsulinType` have hardcoded `label`/`description` constructor properties. These can't use `stringResource()`. Change to resource ID references:

```kotlin
// Before
enum class GlucoseSource(val label: String, val description: String) {
    COMPANION("Companion Mode", "Parse notifications from CGM apps"),
    ...
}

// After
enum class GlucoseSource(@StringRes val labelRes: Int, @StringRes val descriptionRes: Int) {
    COMPANION(R.string.settings_source_companion_label, R.string.settings_source_companion_desc),
    ...
}
```

Callers change from `source.label` to `stringResource(source.labelRes)` (Compose) or `context.getString(source.labelRes)` (non-Compose).

Affected enums:
- `GlucoseSource` — 3 labels + 3 descriptions (6 strings, prefix `settings_source_`)
- `InsulinType` — 4 labels (4 strings, prefix `settings_treatments_`)
- `HbA1cUnit` — labels are "%" and "mmol/mol" (symbols, not translatable — no change needed)

### Theme Mode Persistence

`DisplaySettings.kt` uses `listOf("Light", "Dark", "System")` as both display labels AND persistence keys (compared against stored `themeMode` string). Translating these would break settings.

Fix: use `ThemeMode` enum values for comparison/storage, translate only the display text:

```kotlin
ThemeMode.entries.forEachIndexed { index, mode ->
    SegmentedButton(
        selected = themeMode == mode.name,
        onClick = { onThemeModeChange(mode.name) },
    ) {
        Text(stringResource(mode.labelRes))
    }
}
```

Add `@StringRes val labelRes: Int` to `ThemeMode` enum.

### Notification Channel Names

`AlertManager.createChannels()` skips creation if the channel already exists, meaning channel names/descriptions set at first run are never updated. To support translated channel names:

Change from early-return to always calling `createNotificationChannel()` — Android updates name/description on existing channels without resetting user preferences (importance, sound, vibration).

### Plurals

Use `<plurals>` for countable quantities:

```xml
<plurals name="common_minutes_ago">
    <item quantity="one">%1$d min ago</item>
    <item quantity="other">%1$d min ago</item>
</plurals>
```

Swedish has the same singular/plural structure as English. Access via `resources.getQuantityString(R.plurals.common_minutes_ago, count, count)`.

In practice, most time strings in Strimma use abbreviated forms ("5m", "30s") that don't need pluralization. Full plurals are only needed where we use the word "minute(s)", "reading(s)", etc.

### Non-Translatable Strings

Mark with `translatable="false"`:

```xml
<string name="app_name" translatable="false">Strimma</string>
<string name="common_unit_mmol" translatable="false">mmol/L</string>
<string name="common_unit_mgdl" translatable="false">mg/dL</string>
<string name="common_iob_label" translatable="false">IOB</string>
<string name="common_unit_insulin" translatable="false">U</string>
```

Terms kept in English in all locales: Nightscout, IOB, CGM, BG, SGV, mmol/L, mg/dL, xDrip+, CamAPS, AAPS, GDH, HbA1c, GMI, CV.

## What Stays Hardcoded

- Log messages (`Log.d`, `Log.e`, etc.)
- Exception messages
- API paths (`/api/v1/entries`, `/api/v1/treatments.json`)
- JSON field names (`sgv`, `date`, `direction`)
- Intent actions and extras
- DataStore/SharedPreferences keys
- Package names
- Date/time format patterns (`"yyyy-MM-dd'T'HH:mm:ss"`)

## Contributor Guide

A `TRANSLATING.md` in the repo root:

1. Copy `app/src/main/res/values/strings.xml` to `app/src/main/res/values-XX/strings.xml`
2. Translate values. Keep keys, format placeholders (`%1$s`, `%1$d`), and `<plurals>` structure intact.
3. Don't translate strings marked `translatable="false"`.
4. Don't translate or modify the `name="..."` attributes.
5. Test: change phone language, verify the app uses your strings.
6. Submit a PR.

## Testing

- Verify all `stringResource()` / `context.getString()` calls compile (build catches missing resources)
- Verify all format strings have correct argument count and types
- Manual test: switch device to Swedish, walk through every screen, widget, and trigger each alert type
- Lint: Android lint warns on missing translations and format string mismatches
- Verify notification channel names update on locale change (channels re-created, not skipped)
- Verify Glance widget renders translated strings correctly
- Add a unit test that loads all translation `strings.xml` files and checks every key in default exists in each locale (optional, catches drift)

## Implementation Order

1. Create English `strings.xml` with all extracted strings (organized by prefix)
2. Replace hardcoded strings in Compose UI files with `stringResource()`
3. Replace hardcoded strings in non-Compose files with `context.getString()`
4. Create `values-sv/strings.xml` with Swedish translations
5. Create `TRANSLATING.md`
6. Run Android lint, fix any format string or missing translation warnings
7. Manual verification on device in both locales

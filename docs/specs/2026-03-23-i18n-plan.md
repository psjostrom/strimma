# Strimma i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract ~165 hardcoded strings into Android resources, supporting English (default), Swedish, Spanish, French, and German.

**Architecture:** Standard Android string resources (`strings.xml`). `stringResource()` in Compose, `context.getString()` in services/notifications/Glance. Enum labels change from constructor strings to `@StringRes` resource IDs. Notification channel creation always updates (no early-return skip).

**Tech Stack:** Kotlin, Jetpack Compose, Android Resources, Jetpack Glance

**Spec:** `docs/specs/2026-03-23-i18n-design.md`

### Task 3: Fix AlertManager notification channel updates

**Files:**
- Modify: `app/src/main/java/com/psjostrom/strimma/notification/AlertManager.kt`

The current code skips channel creation if the channel already exists. This means channel names never update when locale changes. Fix: always call `createNotificationChannel()` — Android updates name/description without resetting user preferences.

- [ ] **Step 1: Read AlertManager.kt to find the channel creation pattern**

Read the `createChannels()` method. Look for the early-return pattern:
```kotlin
if (notificationManager.getNotificationChannel(id) != null) return
```

- [ ] **Step 2: Remove the early-return guard from each channel creation**

For each channel in `createChannels()`, remove or relocate the early-return check. The pattern becomes:

```kotlin
// Before
private fun createChannel(id: String, name: String, ...) {
    if (notificationManager.getNotificationChannel(id) != null) return
    val channel = NotificationChannel(id, name, importance)
    channel.description = description
    notificationManager.createNotificationChannel(channel)
}

// After — always create/update
private fun createChannel(id: String, name: String, ...) {
    val existing = notificationManager.getNotificationChannel(id)
    val channel = NotificationChannel(id, name, importance).apply {
        this.description = description
    }
    // Preserve user's sound/vibration preferences from existing channel
    if (existing != null) {
        channel.setSound(existing.sound, existing.audioAttributes)
        channel.enableVibration(existing.shouldVibrate())
        channel.vibrationPattern = existing.vibrationPattern
    }
    notificationManager.createNotificationChannel(channel)
}
```

Note: Actually, `createNotificationChannel()` on an existing channel only updates name and description — it does NOT reset importance, sound, or vibration. So we can simplify: just remove the early-return and always call `createNotificationChannel()`. The user preferences are preserved by Android automatically.

```kotlin
// Simplest correct fix: just remove the early-return
private fun createChannel(id: String, name: String, ...) {
    val channel = NotificationChannel(id, name, importance)
    channel.description = description
    notificationManager.createNotificationChannel(channel)
}
```

- [ ] **Step 1: Create TRANSLATING.md**

```markdown
# Translating Strimma

Strimma uses Android's built-in string resource system for localization.

## Adding a new language

1. Copy `app/src/main/res/values/strings.xml` to `app/src/main/res/values-XX/strings.xml` where `XX` is the [ISO 639-1 language code](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes) (e.g., `values-de` for German, `values-fr` for French).

2. Translate the values between `>` and `</string>`. Keep everything else unchanged:
   - String `name="..."` attributes (the keys)
   - Format placeholders: `%1$s`, `%1$d`, `%1$.1f`
   - Unicode escapes: `\u00B7`, `\u2014`, `\u2026`, etc.

3. Do **not** include strings marked `translatable="false"` in your translation file. These are technical terms and symbols that stay in English for all locales (Nightscout, IOB, mmol/L, etc.).

4. Do **not** translate or modify the `name="..."` attributes.

5. Test your translation: change your phone's language in Settings, then open Strimma and walk through every screen.

6. Submit a pull request.

## Currently supported languages

- English (default)
- Swedish (`values-sv`)
- Spanish (`values-es`)
- French (`values-fr`)
- German (`values-de`)
```
